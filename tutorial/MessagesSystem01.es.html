Sistema de mensajes

Con lluvia podemos pasar mensajes entre aplicaciones. Una aplicación enviará a otra una petición a través de un mensaje. El mensaje emitido quedará almacenado en la cola de la aplicación receptora mientras la otra sigue ejecutando el resto de sus funciones.

INPUT TERMINAL

Your code: 
					systemEv("sync", {name: "activated", 
					                  data: "no extra data available"}, 
                       new Date(),
									  function(p){alert(p.toSource());}
						    )
run

OUTPUT CONSOLE

#debug

Conclusión

El código ejecutado nos muestra un "alert" con el contenido interno del mensaje creado.

El mensaje recibido queda almacenado en la cola. La aplicación receptora emitirá a su vez un "acuse de recibo" informado del id del mensaje recibido a la aplicación emisora.

A través de este método conseguimos que una aplicación no se quede "parada" hasta que no se ejecute el evento solicitado (supongamos que la petición fuera calcular número pi... la aplicación que solicita el evento quedaría esperando eternamente hasta volver a realizar sus propias funciones).
